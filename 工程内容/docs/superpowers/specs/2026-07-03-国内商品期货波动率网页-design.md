# 国内商品期货波动率网页设计

## 目标

做一个可联网访问的国内商品期货波动率网页。网页第一版以静态站点形式运行，数据由定时脚本更新。用户拿到公开链接后，只要能联网，就可以查看最近一次更新后的国内商品期货波动率表。

## 范围

- 覆盖国内商品期货主力或连续合约。
- 不覆盖股指期货、国债期货和期权。
- 第一版展示日频历史波动率，不做分钟级实时波动率。
- 第一版每天收盘后更新一次；夜盘后补更新作为后续扩展。
- 第一版实现本地可预览、仓库可部署到 GitHub Pages 的完整工程。

## 数据口径

数据生成脚本获取每个品种的日线数据，字段至少包含日期、开盘、最高、最低、收盘、成交量。计算口径如下：

- 日收益率：`ln(close_t / close_t-1)`。
- 20 日年化波动率：最近 20 个收益率标准差乘以 `sqrt(252)`。
- 60 日年化波动率：最近 60 个收益率标准差乘以 `sqrt(252)`。
- 当日涨跌幅：`close_t / close_t-1 - 1`。
- 当日振幅：`(high_t - low_t) / close_t-1`。
- 波动率分位：当前 20 日年化波动率在最近 252 个交易日滚动 20 日波动率序列中的百分位。

若某合约历史长度不足，脚本保留该行并在 `status` 中标记数据不足，而不是让品种消失。

## 架构

项目由三部分组成：

1. Python 数据层：负责获取日线数据、计算指标、生成 `site/data/volatility.json`。
2. 静态网页层：`site/index.html`、`site/styles.css`、`site/app.js` 读取 JSON 并渲染可筛选、可排序表格。
3. 部署自动化层：GitHub Actions 每天定时运行数据脚本，再把 `site` 目录发布到 GitHub Pages。

网页不依赖后端服务，因此别人访问时只需要打开公开链接。数据更新失败时，页面继续显示上一份可用快照，并在状态区提示更新时间和数据源状态。

## 页面设计

首页直接展示工具界面，不做营销落地页。顶部是标题、更新时间、数据日期、数据源状态和汇总卡片。中部是搜索框、交易所筛选、排序控件和刷新提示。主体是密集但清晰的数据表，字段包括：

- 交易所
- 品种
- 合约
- 最新日期
- 收盘价
- 涨跌幅
- 当日振幅
- 20 日波动率
- 60 日波动率
- 波动率分位
- 状态

视觉风格偏交易工具：高信息密度、浅色背景、低饱和强调色、移动端可横向浏览表格。

## 错误处理

- 数据文件加载失败：页面显示错误提示和内置样例数据不可用提示。
- 单个品种抓取失败：脚本记录到元数据 `errors`，页面不显示该失败品种的错误堆栈，只显示总错误数量。
- 数据不足：保留品种行，状态显示“数据不足”。
- 计算出现无效价格：跳过无效日期，若有效样本不足则标记数据不足。

## 测试

- Python 单元测试覆盖收益率、年化波动率、振幅、分位数、数据不足状态。
- JavaScript 单元测试覆盖筛选、排序、格式化、表格渲染 HTML。
- 本地验证使用内置 Python HTTP 服务启动 `site` 目录，使用浏览器或 HTTP 请求确认页面和 JSON 可加载。

## 部署

仓库包含 `.github/workflows/update-and-deploy.yml`。工作流在北京时间收盘后运行，对应 UTC 时间约 08:45。工作流步骤：

1. 安装 Python 依赖。
2. 运行测试。
3. 运行数据生成脚本。
4. 上传 `site` 目录为 GitHub Pages artifact。
5. 部署到 GitHub Pages。

如果本机没有 GitHub 登录状态，交付时提供完整工程和部署说明，由用户推送到自己的 GitHub 仓库后启用 Pages。

## 验收标准

- 本地 `site/index.html` 能通过 HTTP 服务打开。
- 页面能读取 `site/data/volatility.json` 并展示波动率表。
- 数据 JSON 包含元数据、汇总和合约行。
- Python 和 JavaScript 测试通过。
- 仓库包含可用于 GitHub Pages 的部署工作流。
