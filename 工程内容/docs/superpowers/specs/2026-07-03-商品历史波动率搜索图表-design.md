# 商品历史波动率搜索图表设计

## 目标

把现有“60 个商品波动率表格”改成“搜索选择商品 + 日期区间 + 单商品历史图表”的模式。网页初始状态不铺开每个商品的具体波动率行，只提供商品搜索/选择入口。用户选择某个商品后，展示该商品从上市以来到最新日期的历史行情和滚动波动率，并允许按日期区间筛选，例如 2026-01-03 到 2026-06-03。

## 数据结构

`site/data/volatility.json` 从快照表升级为历史数据包：

- `meta`：生成时间、数据日期、数据源、错误信息。
- `summary`：商品总数、正常数量、失败数量。
- `contracts`：60 个商品的选择器数据，包含交易所、中文名、合约代码、首日、最新日、状态。
- `series`：以合约代码为键，每个值是一组日线历史数据。

每条历史数据包含：

- `date`
- `open`
- `high`
- `low`
- `close`
- `change_pct`
- `amplitude_pct`
- `vol_20`
- `vol_60`

滚动波动率从有足够样本的日期开始出现；样本不足的早期日期保留价格 K 线但波动率字段为 `null`。

## 页面交互

- 顶部显示数据日期、更新时间、数据源。
- 商品选择器支持输入搜索和下拉选择，选项为 60 个商品。
- 默认选择第一个正常商品，但不展示 60 行表格。
- 日期区间输入框根据所选商品自动设置最早和最新可选日期。
- 点击或修改日期后，图表只展示区间内数据。
- 提供快捷按钮：近 3 个月、近 6 个月、近 1 年、全部。

## 图表设计

使用原生 Canvas 实现，不引入外部图表库，保证 GitHub Pages 静态站可直接运行。

- 上半区：价格 K 线，使用开高低收绘制蜡烛。
- 下半区：20 日和 60 日年化波动率曲线。
- 鼠标移动显示十字光标和当前日期、OHLC、20 日/60 日波动率。
- 日期区间过窄时仍能显示，空区间显示提示。

## 更新与发布

数据生成脚本继续使用 AKShare `futures_zh_daily_sina` live 模式获取完整历史日线。Codex 本地定时任务和 GitHub Pages 发布方式保持不变，但发布的数据文件变为历史数据包。

## 验收标准

- 公网页面初始不列出 60 行商品波动率表。
- 页面有商品搜索/选择框，包含 60 个商品。
- 选中商品后显示该商品历史 K 线图和 20 日/60 日波动率曲线。
- 日期区间能筛选图表数据。
- 数据历史覆盖到该商品数据源返回的最早日期和最新日期。
- Python 和 JavaScript 测试通过。
- GitHub Pages 公网链接可访问新版页面。
