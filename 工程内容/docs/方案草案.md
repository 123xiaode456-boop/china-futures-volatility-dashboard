# 国内商品期货波动率网页方案草案

## 目标

做一个可联网访问的网页，用于每天查看国内商品期货波动率。用户把网页链接发给别人后，对方只要能联网，就能查看最新更新后的页面。

## 默认产品口径

- 第一版默认展示每个商品期货品种的主力或连续合约，适合快速扫全市场。
- 页面设计预留“展开全部合约”的数据结构，但第一版不强制做全部到期月份合约。
- 覆盖范围以国内商品期货为主，不包含股指期货、国债期货、期权。

## 默认波动率口径

- 使用日频收盘价计算历史波动率。
- 20 日年化波动率：最近 20 个交易日对数收益率标准差乘以 `sqrt(252)`。
- 60 日年化波动率：最近 60 个交易日对数收益率标准差乘以 `sqrt(252)`。
- 当日振幅：`(最高价 - 最低价) / 昨收或前收盘价`，如数据源没有昨收则用前一交易日收盘价。
- 波动率分位：当前 20 日波动率在近一年历史 20 日波动率序列中的百分位。

## 推荐技术路线

首选方案：GitHub Pages + GitHub Actions + Python 数据脚本。

- GitHub Actions 每个交易日收盘后定时运行。
- Python 脚本通过 AKShare 获取国内商品期货日线数据。
- 脚本计算各品种波动率指标，输出静态 JSON 数据文件。
- GitHub Pages 托管一个静态网页，网页读取 JSON 后渲染表格。
- 公开链接可分享给别人访问，不需要别人安装软件。

## 备选方案

### Vercel 或 Cloudflare + Serverless

优点是更像实时服务，可以按访问动态刷新。缺点是后端接口、调用额度、缓存和错误处理更复杂。

### 本机脚本 + 本机网页

优点是原型最快。缺点是别人不能稳定访问，不满足“发给别人，对方联网即可查看”的目标。

## 页面功能

- 顶部显示最新数据日期、更新时间、数据源状态。
- 主表字段：交易所、品种、合约、最新日期、收盘价、涨跌幅、当日振幅、20 日波动率、60 日波动率、波动率分位。
- 支持搜索品种或合约。
- 支持按交易所筛选。
- 支持按 20 日波动率、60 日波动率、波动率分位排序。
- 数据不足或获取失败的品种显示明确状态，不静默消失。

## 数据与更新

- 每天收盘后定时更新一次，建议北京时间 16:30 后执行。
- 如果夜盘数据需要纳入，后续可增加夜盘后补更新。
- 数据文件保留最近一次完整快照，并保留原始抓取日志，便于排查。
